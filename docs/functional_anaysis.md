# 기능 분석

## 주소록

### 모델링
- User(연락처를 가지고 있는 사용자)
- ContactBook(주소록)
- Label(라벨)
- ContactLabel(라벨링 된 연락처 관리)


### 기능 요구사항

- 목록 조회
  - [ ] 조회시 출력할 필드가 모두 있다.
      - [ ] 프로필 사진
      - [ ] 이름
      - [ ] 이메일
      - [ ] 전화번호
      - [ ] 회사(직책)
      - [ ] 라벨
  - [ ] 정렬
    - [ ] 기본 출력은 등록 순서대로 정렬 한다.
    - [ ] 이름, 이메일, 전화번호 중
    - [ ] 정렬은 오름차순 / 내림차순 / 해제 순이다.
  - [ ] 페이징
    - [ ] 스크롤 페이징 처리가 된다.
- 상세 보기 / 입력
  - [ ] 입력과 출력시 다음과 같은 필드를 입력 받을 수 있다.
      - [ ] 프로필 사진 : url 입력 방식
      - [ ] 이름
      - [ ] 이메일
      - [ ] 전화번호
      - [ ] 회사(필수 아님)
      - [ ] 직책(필수 아님)
      - [ ] 메모(필수 아님)
      - [ ] 라벨(필수 아님)
        - [ ] 사용자 정의 라벨
        - [ ] 연락처 1개에 라벨 다수 연결 가능
      - [ ] 기타 항목 추가(필수 아님)
        - [ ] 주소
        - [ ] 생일
        - [ ] 웹사이트
